# Order Service - 订单服务

电商系统中的订单管理微服务，基于 Go-Kratos 框架构建。

## 业务功能

### 订单管理
- **创建订单**：根据用户、收货地址和商品列表创建订单
- **查询订单**：根据订单 ID 获取订单详情
- **查询订单列表**：分页查询用户的订单列表
- **更新订单状态**：修改订单状态（已创建→已支付→已发货→已完成）
- **取消订单**：取消已创建的订单

## 订单状态流转

```
已创建 → 已支付 → 已发货 → 已完成
  ↓
已取消
```

| 状态 | 说明 |
|------|------|
| 已创建 | 订单已创建，等待支付 |
| 已支付 | 用户已完成支付 |
| 已发货 | 商家已发货 |
| 已完成 | 用户已收货 |
| 已取消 | 订单已取消 |

## 订单事件

订单服务支持事件驱动架构，发布以下事件：

| 事件类型 | 说明 |
|----------|------|
| 订单创建事件 | 新订单创建时发布 |
| 订单更新事件 | 订单状态变更时发布 |
| 订单取消事件 | 订单取消时发布 |

## 订单信息

| 字段 | 说明 |
|------|------|
| 订单 ID | 唯一标识 |
| 用户 ID | 下单用户 |
| 订单状态 | 当前订单状态 |
| 订单项 | 商品列表（商品 ID、名称、数量、单价） |
| 总金额 | 订单总金额（分） |
| 收货地址 | 收货地址 ID |

## 技术栈

- **框架**: Go-Kratos
- **通信**: gRPC
- **数据库**: GORM
- **语言**: Go
